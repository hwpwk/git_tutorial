ローカルは3つのエリアに分かれれている
ワークツリー→ステージ→リポリトジ
ワークツリー：ファイルを変更する作業場
作業場で変更したらそれをリポリトジに上げるのだが、その前にステージに変更するという作業が必要
ステージ：コミットする変更を準備
commit→スナップショットを記録すること
gitでは変更をいったんステージに記録してからリポリトジに上げる仕組みになっている
この際、つまりワークツリーから→ステージに一部の変更を記録する際に「git add」というコマンドを実行する（ワークツリー→ステージ）
ステージに変更を記録した後に「git commit」コマンドでリポリトジにスナップショットを記録する（ステージ→リポリトジ）
これがローカルでの作業全体となる

ワークツリー
↓　git add で変更を記録
ステージ
↓　git commitでスナップショットを記録
リポリトジ



git commit -m '<メッセージ>'
→gitのエディタを立ち上げることなくメッセージ付きで変更を記録できる
git commit -v
→gitエディタが立ち上がりそこにメッセージを付ける

git remote add origin https://~
→originというショートカットでURLのリモートリポジトリを登録する、という意味
→originと指定することで今後はoriginという名前でGitHubリポジトリにアップしたり取得したりできる

git push -u origin mater
→「-u」オプションを付けておくことで次回以降「git push」コマンドのみでorigin materをプッシュできる

エイリアス→別名のこと

touch secret.txt
→「touch」:空ファイルを作ること

git commit --amend
→直前のコミットをやり直す
